Saya ingin melakukan refactor total terhadap project Tattoo Studio Management System yang saat ini masih berupa project Flask sederhana. Tujuan refactor bukan hanya merapikan kode, tetapi membangun ulang arsitektur project agar mengikuti standar Enterprise Application menggunakan pola **MVC (Model–View–Controller)** dengan prinsip Clean Architecture, SOLID Principle, Separation of Concerns, serta Best Practice Flask. Backend tetap menggunakan Flask dan MongoDB, sedangkan frontend tetap menggunakan Jinja2, HTML, CSS, dan JavaScript. Jangan mengubah fitur yang sudah ada, tetapi ubah struktur project agar lebih profesional, scalable, aman, dan siap dikembangkan menjadi aplikasi SaaS.

Pisahkan project menjadi dua bagian yang jelas, yaitu **Frontend** dan **Backend**. Seluruh kode yang berhubungan dengan tampilan seperti HTML, CSS, JavaScript, Font, Image, Asset, dan Template harus berada pada folder frontend. Seluruh kode yang berhubungan dengan business logic, database, authentication, middleware, API, service, model, repository, helper, validation, security, logging, dan konfigurasi harus berada pada folder backend. Saya ingin struktur folder yang mudah dipahami oleh developer lain dan mengikuti standar industri.

Bangun ulang struktur project menggunakan pola MVC sehingga setiap fitur memiliki Controller, Service, Repository, Model, dan Route yang terpisah. Jangan lagi meletakkan seluruh route di satu file run.py. Semua route harus dipisahkan berdasarkan modul seperti Authentication, Artist, Booking, Customer, Gallery, Payment, Dashboard, Admin, Service, dan API. Setiap Controller hanya bertugas menerima request dan mengembalikan response. Seluruh business logic harus dipindahkan ke Service Layer. Seluruh akses database harus dipindahkan ke Repository Layer. Model hanya bertanggung jawab terhadap struktur data dan validasi dasar.

Gunakan Flask Blueprint untuk memisahkan setiap modul. Jangan menggunakan global variable. Semua konfigurasi aplikasi harus dipindahkan ke config.py dan menggunakan environment variable (.env). Jangan ada secret key, database URI, credential WhatsApp, ataupun API Key yang ditulis langsung di source code. Gunakan python-dotenv agar seluruh konfigurasi berasal dari environment.

Implementasikan keamanan dengan standar tinggi. Gunakan password hashing menggunakan Werkzeug Security atau Argon2/Bcrypt. Gunakan CSRF Protection pada seluruh form. Gunakan Flask-WTF apabila diperlukan. Implementasikan Rate Limiting pada endpoint login menggunakan Flask-Limiter. Tambahkan validasi input pada seluruh request agar terhindar dari Injection dan Invalid Data. Seluruh data yang masuk dari user harus divalidasi terlebih dahulu sebelum diproses. Jangan mempercayai data dari frontend.

Bangun sistem autentikasi menggunakan Session Authentication yang aman. Tambahkan Role Based Access Control (RBAC) dengan role seperti Super Admin, Admin, Artist, Staff, dan Customer. Setiap route harus memiliki middleware yang memeriksa apakah user sudah login dan memiliki hak akses yang sesuai. Jangan hanya menyembunyikan menu di frontend, tetapi pastikan backend juga melakukan pengecekan authorization.

Pisahkan middleware ke folder tersendiri, seperti Authentication Middleware, Authorization Middleware, Logging Middleware, dan Error Handling Middleware. Seluruh exception harus ditangani secara terpusat sehingga tidak menampilkan traceback kepada user ketika mode production.

Buat folder khusus untuk utility seperti helper, formatter, validator, constants, decorator, logger, email service, WhatsApp service, storage service, dan security helper. Jangan menuliskan fungsi utilitas secara acak di dalam controller.

Pisahkan frontend menjadi struktur yang rapi. Template dibagi menjadi layout, partial, component, page, admin, customer, auth, booking, payment, gallery, artist, dan dashboard. CSS dipisahkan menjadi base, layout, component, utilities, pages, dan variables. JavaScript dipisahkan berdasarkan modul seperti booking, authentication, dashboard, payment, gallery, artist, admin, dan shared component.

Gunakan struktur folder yang bersih seperti backend/app/controllers, backend/app/services, backend/app/repositories, backend/app/models, backend/app/routes, backend/app/middlewares, backend/app/config, backend/app/extensions, backend/app/utils, backend/app/security, backend/app/database, backend/app/schemas, backend/app/exceptions, backend/app/logs, backend/app/tests, serta frontend/templates, frontend/static/css, frontend/static/js, frontend/static/images, frontend/static/fonts, frontend/static/icons, frontend/static/uploads, frontend/static/vendor, frontend/components, frontend/layouts, dan frontend/pages.

Tambahkan logging menggunakan Python Logging dengan Rotating File Handler sehingga seluruh error, warning, dan activity dapat tercatat pada folder logs. Tambahkan Global Error Handler sehingga seluruh response error memiliki format yang konsisten.

Buat seluruh kode mudah dipelihara dengan memberikan komentar seperlunya, menggunakan type hint, penamaan variabel yang konsisten, serta mengikuti standar PEP8. Jangan membuat file yang terlalu besar; apabila sebuah file melebihi sekitar 300–500 baris, pecah menjadi beberapa modul yang lebih kecil.

Pastikan struktur project yang dihasilkan siap dikembangkan menjadi SaaS multi-tenant di masa depan. Oleh karena itu, arsitektur harus modular, mudah diuji (testable), mudah diperluas, dan mudah dipindahkan ke Docker maupun deployment cloud seperti VPS, Railway, Render, atau AWS.

Terakhir, lakukan refactor secara menyeluruh. Jangan hanya memindahkan file, tetapi sesuaikan seluruh import, Blueprint, routing, konfigurasi, template rendering, static file, business logic, dependency, dan struktur folder agar project dapat langsung dijalankan tanpa error. Berikan penjelasan untuk setiap perubahan arsitektur yang dilakukan beserta alasan teknisnya, dan tampilkan struktur folder akhir secara lengkap sehingga project siap digunakan sebagai pondasi aplikasi Tattoo Studio Management System yang profesional dan aman.
